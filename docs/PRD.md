# 产品需求文档 (PRD) - HaliClip 剪贴板助手

## 1. 项目背景
在日常办公和开发过程中，频繁的复制粘贴是核心操作。系统自带剪贴板仅支持单条记录，且在重启后丢失。HaliClip 旨在提供一个轻量级、跨平台、无感化的剪贴板历史管理工具。

## 2. 目标用户
- 软件开发人员
- 文案策划与编辑
- 重度办公辅助工具使用者

## 3. 核心功能需求

### 3.1 全局无感运行
- **启动模式**：应用启动时不显示主窗口，仅在系统托盘显示图标。
- **托盘交互**：
  - 左键/右键点击弹出菜单。
  - 菜单项：显示历史、设置、关于、开机自启开关、退出。

### 3.2 剪贴板历史界面 (Maccy Style)
- **极简 UI**：无标题栏窗口，类似弹出式菜单，紧凑展示历史项。
- **呼出位置**：默认在鼠标当前光标位置弹出（可选屏幕中央）。
- **搜索优先**：呼出后焦点立即落在搜索框（或直接监听键盘输入进行过滤）。
- **键盘操作 (Power User)**：
  - `Cmd/Alt + [1-9]`：直接应用前 9 条记录。
  - `Up/Down`：循环切换选中项。
  - `Enter`：将选中项内容写回剪贴板并尝试执行粘贴（Cmd+V 模拟）。
  - `Shift + Enter`：仅复制，不隐藏窗口。
  - `Cmd/Alt + P`：固定（Pin）某条记录。
  - `Cmd/Alt + Delete`：删除该条记录。

### 3.3 全局快捷键
- **功能触发**：默认 `Cmd + Shift + C` (macOS) 或 `Alt + Shift + V` (Windows/Linux)。
- **一致性**：快捷键逻辑需高度模仿 Maccy，确保老用户无缝迁移。

### 3.4 设置中心
- **基础设置**：开机自启动。
- **历史策略**：最大存储条数（如 50/100/500 条）。
- **快捷键配置**：用户可自定义呼出快捷键。

## 4. 权限与安全需求
- **macOS 辅助功能权限**：全局快捷键监听在 macOS 上需要用户授权“辅助功能 (Accessibility)”。应用需在未获得权限时引导用户开启。
- **剪贴板访问权限**：在沙盒环境下需配置相应的 Entitlements。
- **隐私保护**：所有剪贴板内容仅在本地存储，严禁在未经用户允许的情况下上传至任何服务器。

## 5. 非功能需求
- **跨平台一致性**：支持 Windows 10/11, macOS (Intel/Apple Silicon), Linux (Ubuntu/Fedora)。
- **性能优化**：数据库读写不阻塞 UI，内存占用需控制在 100MB 以内。
- **交互体验**：窗口呼出需平滑，支持失去焦点自动隐藏。

## 5. 成功指标
- 剪贴板监听延迟 < 100ms。
- 窗口呼出速度 < 200ms。
- 单个安装包体积 < 50MB。
