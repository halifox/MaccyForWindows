# Maccy v2.6.1 架构深度分析与 Windows Flutter 实现方案

## 一、数据库结构（Single Source of Truth）

### 1.1 核心数据模型

Maccy 使用 SwiftData (Core Data) 存储，包含两个核心实体：

#### **HistoryItem** (历史条目主表)
```swift
@Model
class HistoryItem {
  var application: String?           // 来源应用 bundle ID
  var firstCopiedAt: Date            // 首次复制时间
  var lastCopiedAt: Date             // 最后复制时间
  var numberOfCopies: Int = 1        // 复制次数
  var pin: String?                   // 固定快捷键 (b-y, 排除 a/q/v/w/z)
  var title: String = ""             // 显示标题 (最大 203 字符)
  
  @Relationship(deleteRule: .cascade)
  var contents: [HistoryItemContent] // 一对多关系，级联删除
}
```

#### **HistoryItemContent** (内容数据表)
```swift
@Model
class HistoryItemContent {
  var type: String = ""              // 内容类型 (NSPasteboard.PasteboardType)
  var value: Data?                   // 二进制数据
  
  @Relationship
  var item: HistoryItem?             // 反向关系
}
```

### 1.2 您的 Flutter 实现对比

**✅ 完全匹配 Maccy 原生结构**

## 二、核心模块功能分解

### 2.1 剪贴板监控模块 (Clipboard Monitoring)

**Maccy 实现：**
- **文件**: `Clipboard.swift`
- **核心机制**:
  - 使用 `Timer` 定时轮询 `NSPasteboard.general.changeCount`
  - 默认轮询间隔: `0.5秒` (可配置 `clipboardCheckInterval`)
  - 支持的类型: `.fileURL`, `.html`, `.png`, `.rtf`, `.string`, `.tiff`
  - 忽略类型: `.autoGenerated`, `.concealed`, `.transient`

**关键逻辑**:
```swift
// Clipboard.swift:150-224
@objc @MainActor
func checkForChangesInPasteboard() {
  guard pasteboard.changeCount != changeCount else { return }
  changeCount = pasteboard.changeCount
  
  // 1. 检查是否忽略事件
  if Defaults[.ignoreEvents] { return }
  
  // 2. 过滤类型和应用
  if shouldIgnore(Set(pasteboard.types ?? [])) { return }
  if shouldIgnore(sourceApp?.bundleIdentifier) { return }
  
  // 3. 合并多个 pasteboardItems 到单个 HistoryItem
  var contents = [HistoryItemContent]()
  pasteboard.pasteboardItems?.forEach { item in
    // 过滤空字符串、动态类型、Microsoft 特殊类型
    types.forEach { type in
      contents.append(HistoryItemContent(type: type.rawValue, value: item.data(forType: type)))
    }
  }
  
  // 4. 创建 HistoryItem 并触发回调
  let historyItem = HistoryItem(contents: contents)
  historyItem.application = sourceApp?.bundleIdentifier
  historyItem.title = historyItem.generateTitle()
  onNewCopyHooks.forEach({ $0(historyItem) })
}
```

**Windows Flutter 实现要点**:
- 使用 `clipboard_watcher` 包监听剪贴板变化
- 使用 `super_clipboard` 读取多格式数据
- 实现 `foreground_app_service.dart` 获取前台应用信息
- 实现 `clipboard_filter_service.dart` 过滤逻辑

### 2.2 历史管理模块 (History Management)

**Maccy 实现：**
- **文件**: `Observables/History.swift`
- **核心职责**:
  1. 管理 `all` (所有条目) 和 `items` (可见条目) 两个列表
  2. 去重逻辑：检测相同内容，合并 `numberOfCopies` 和时间戳
  3. 排序：支持 `lastCopiedAt`, `firstCopiedAt`, `numberOfCopies` 三种排序
  4. Pin 管理：固定条目始终在顶部/底部（可配置 `pinTo`）
  5. 搜索：实时过滤 `items`，使用 `Throttler` 防抖（0.2秒）

**关键方法**:
```swift
// History.swift:141-202
@MainActor
func add(_ item: HistoryItem) -> HistoryItemDecorator {
  // 1. 查找重复项
  if let existingHistoryItem = findSimilarItem(item) {
    item.firstCopiedAt = existingHistoryItem.firstCopiedAt
    item.numberOfCopies += existingHistoryItem.numberOfCopies
    item.pin = existingHistoryItem.pin
    Storage.shared.context.delete(existingHistoryItem)
  }
  
  // 2. 限制历史大小
  limitHistorySize(to: Defaults[.size] - 1)
  
  // 3. 插入到正确位置（根据排序规则）
  let sortedItems = sorter.sort(all.map(\.item) + [item])
  if let index = sortedItems.firstIndex(of: item) {
    all.insert(itemDecorator, at: index)
  }
  
  return itemDecorator
}
```

**Windows Flutter 实现要点**:
- 在 `clipboard_manager_provider.dart` 中实现去重逻辑
- 使用 Riverpod 状态管理 `all` 和 `filtered` 列表
- 实现 `search_service.dart` 和 `advanced_search_service.dart`

### 2.3 搜索模块 (Search)

**Maccy 实现：**
- **文件**: `Search.swift`
- **支持模式**:
  1. **Exact**: 大小写不敏感的精确匹配
  2. **Fuzzy**: 使用 Fuse 库（阈值 0.7），限制 5000 字符
  3. **Regexp**: 正则表达式搜索
  4. **Mixed**: 依次尝试 Exact → Regexp → Fuzzy

**关键代码**:
```swift
// Search.swift:39-54
func search(string: String, within: [Searchable]) -> [SearchResult] {
  guard !string.isEmpty else {
    return within.map { SearchResult(object: $0) }
  }
  
  switch Defaults[.searchMode] {
  case .mixed:
    return mixedSearch(string: string, within: within)
  case .regexp:
    return simpleSearch(string: string, within: within, options: .regularExpression)
  case .fuzzy:
    return fuzzySearch(string: string, within: within)
  default:
    return simpleSearch(string: string, within: within, options: .caseInsensitive)
  }
}
```

**Windows Flutter 实现要点**:
- 使用 `fuzzy` 包实现模糊搜索
- 在数据库层注册 `regexp` 函数（已完成）
- 实现 `advanced_search_service.dart` 支持多模式切换

### 2.4 排序模块 (Sorter)

**Maccy 实现：**
- **文件**: `Sorter.swift`
- **排序规则**:
  1. **主排序**: Pin 条目始终在顶部/底部（根据 `pinTo` 配置）
  2. **次排序**: 根据 `sortBy` 配置
     - `lastCopiedAt`: 最后复制时间降序（默认）
     - `firstCopiedAt`: 首次复制时间降序
     - `numberOfCopies`: 复制次数降序

```swift
// Sorter.swift:26-30
func sort(_ items: [HistoryItem], by: By = Defaults[.sortBy]) -> [HistoryItem] {
  return items
    .sorted(by: { return bySortingAlgorithm($0, $1, by) })
    .sorted(by: byPinned)  // Pin 条目优先
}
```

### 2.5 弹窗管理模块 (Popup Management)

**Maccy 实现：**
- **文件**: `FloatingPanel.swift`, `Observables/Popup.swift`
- **核心特性**:
  1. **NSPanel** 浮动窗口，级别 `.statusBar`
  2. **三种弹出模式**:
     - **Toggle**: 快捷键切换显示/隐藏

     - `center`: 屏幕中心
     - `window`: 自定义位置（可拖动保存）
  4. **自动调整高度**: 根据内容动态调整，最大不超过 `windowSize.height`

**关键代码**:
```swift
// FloatingPanel.swift:65-85
func toggle(height: CGFloat, at popupPosition: PopupPosition) {
```

### 2.7 粘贴模块 (Paste Service)

**Maccy 实现：**
- **文件**: `Clipboard.swift:112-138`
- **核心机制**:
  1. 使用 `CGEvent` 模拟 `Cmd+V` 按键
  2. 检查辅助功能权限 (`Accessibility.check()`)
  3. 处理特殊键盘布局（如 Dvorak - QWERTY ⌘）
  4. 禁用本地键盘事件过滤，防止干扰

```swift
// Clipboard.swift:112-138
func paste() {
  Accessibility.check()
  
  let cmdFlag = CGEventFlags(rawValue: UInt64(KeyChord.pasteKeyModifiers.rawValue) | 0x000008)
  var vCode = Sauce.shared.keyCode(for: KeyChord.pasteKey)
  
  // 处理特殊键盘布局
  if KeyboardLayout.current.commandSwitchesToQWERTY && cmdFlag.contains(.maskCommand) {
    vCode = KeyChord.pasteKey.QWERTYKeyCode
  }
  
  let source = CGEventSource(stateID: .combinedSessionState)
  source?.setLocalEventsFilterDuringSuppressionState([.permitLocalMouseEvents, .permitSystemDefinedEvents],
                                                     state: .eventSuppressionStateSuppressionInterval)
  
  let keyVDown = CGEvent(keyboardEventSource: source, virtualKey: vCode, keyDown: true)
  let keyVUp = CGEvent(keyboardEventSource: source, virtualKey: vCode, keyDown: false)
  keyVDown?.flags = cmdFlag
  keyVUp?.flags = cmdFlag
  keyVDown?.post(tap: .cgSessionEventTap)
  keyVUp?.post(tap: .cgSessionEventTap)
}
```

**Windows Flutter 实现要点**:
- 使用 Win32 API `SendInput` 模拟 `Ctrl+V`
- 实现 `paste_service.dart` 封装粘贴逻辑
- 需要处理不同应用的粘贴延迟问题

## 六、关键差异与注意事项

### 6.1 平台差异

| 特性 | macOS (Maccy) | Windows (Flutter) |
|-----|--------------|------------------|
| 剪贴板 API | NSPasteboard | Win32 Clipboard API |
| 全局热键 | KeyboardShortcuts | hotkey_manager + Win32 |
| 浮动窗口 | NSPanel | window_manager |
| 应用识别 | Bundle ID | 进程名/可执行文件路径 |
| 粘贴模拟 | CGEvent | SendInput |
| 系统托盘 | NSStatusItem | tray_manager |

### 6.2 需要特别注意的逻辑

1. **去重算法**: Maccy 使用 `supersedes()` 方法比较内容，需要精确复刻
2. **Pin 快捷键**: 排除 `a/q/v/w/z` 五个字符
3. **搜索防抖**: 使用 0.2 秒延迟，避免频繁查询
4. **窗口高度**: 动态计算，最大不超过配置的 `windowSize.height`
5. **Cycle 模式**: 需要监听修饰键释放事件，这是 Maccy 的核心交互特性

### 6.3 性能优化建议

1. **数据库索引**: 为 `lastCopiedAt`, `pin`, `title` 添加索引
2. **图片缓存**: 大图片使用缩略图缓存
3. **搜索优化**: Fuzzy 搜索限制 5000 字符（与 Maccy 一致）
4. **内存管理**: 限制历史记录数量（默认 200）
